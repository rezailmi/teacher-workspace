# PG Portal — What's Not Done

**Date:** 2026-03-30
**Source spec:** `plans/PG-specs.md`
**Against:** `web/` codebase on `feat/prod-scaffold`

> **Note:** Route paths do NOT need to match the PG spec. TW uses its own route conventions (RFC-027). Path mismatches are intentional and excluded from this gap list.

## 1. MEETINGS (PTM) — 0% complete

The entire module is absent. No routes, no components, no mock data.

### Routes to build

- `web/routes/meetings.tsx` — layout wrapper
- `web/routes/meetings.index.tsx` — list page
- `web/routes/meetings.new.tsx` — 4-step creation wizard
- `web/routes/meetings.$eventId.tsx` — detail page

### 1a. Meetings List (`/meetings`)

- Page header "Meetings" + "+ New meeting event" button
- Two sections: **Upcoming** and **Past**
- Each event row renders:
  - Date badge(s): month abbreviation + day (e.g. "MAR 26"), or a range
  - Title (bold)
  - Sub-info row: calendar icon + N meeting days, clock icon + N min per slot, grid icon + N slots per timing
  - Status line: "Booking period: [date range]" or "Booking period has ended"
  - Status badge: Upcoming | Booking Open | Booking Closed | Past
  - Stats: Available count, ✓ Booked, ? Pending, 👥 Total students
- Row click → `/meetings/details/:eventId`
- Kebab per row: View, Edit, Delete

### 1b. Create Meeting Wizard (`/meetings/new`)

4-step stepper: Basic Info → Meeting Details → Booking Period → Preview

**Step 1 — Basic Info:**

- Recipients (Parents): same student groups combobox as Announcements
- Staff-in-charge (optional): same staff selector
- Enquiry Email (required): same radio options
- Content: Title (120 char), Description (2000 char, rich text), Venue (120 char), Website links (up to 3), File attachments (up to 3, 5MB each)
- "Next" button

**Step 2 — Meeting Details:**

- Meeting duration dropdown (10 min–720 min in 5-min increments)
- Max meetings per slot: radio cards — "One" | "Two" | "Three"
- Meeting days: repeatable rows — date picker + From time + To time (10-min increment dropdown) + trash icon
- "+ Add day" button
- Live-updating summary box: "There will be N meetings available for booking by the parents of M students"
- "Back" | "Next"

**Step 3 — Booking Period:**

- Read-only summary card: lists each meeting day with date/time
- Booking opens: date picker + time dropdown (30-min increments 12:00AM–11:30PM)
- Booking closes: date picker + time dropdown (same increments; dates up to meeting day only)
- Default reminders (auto-computed, read-only): shows "-" until dates filled
- "Back" | "Preview"

**Step 4 — Preview:**

- Carousel with 4 slides and dot pagination + left/right arrows:
  1. "On publish" — push notification mockup + phone app mockup with "NOT STARTED" badge
  2. "Booking opens" — "BOOKING OPEN" badge on phone mockup
  3. "Booking closes" — appointment details on phone mockup, "BOOKING CLOSED" badge
  4. "Meeting starts" — illustration of two people + day card overlay
- "Back" | "Publish"
- Quick tips modal on first load ("You can create meeting event: for meetings on X onwards, that parents can book from Y onwards")

### 1c. Meeting Detail Page (`/meetings/details/:eventId`)

**Schedule Tab (default):**

- "Total of N meeting day(s)"
- Day selector with left/right navigation
- "Export to Excel" button
- Status line: "You have N confirmed meetings on [Bold Date] (as at HH:MM AM)"
- Timeslot list: each row shows time + booked parent names or "Available"
  - ⊕ icon → "Add/edit details for slot" modal (student search + parent comments textarea + Cancel/Save)
  - ⊘ icon → immediately toggles to "Blocked" state (clicking again unblocks)

**Details Tab:**

- Posted date/time/author line
- Meeting details table: date(s) & time, duration, meetings per slot, available slots, booking period, default reminders
- Staff-in-charge + "+ Add" modal
- Enquiry Email + "Edit" modal
- OTHER ACTIONS: "Delete this meeting event" → delete confirmation modal

## 3. ANNOUNCEMENTS — Gaps (~15%)

Core flow exists. Gaps are:

### 3a. Separate routes missing

- `/announcements/preview` — phone mockup preview page. Currently the "Preview" button in the create/edit form goes nowhere in a separate route. Spec says it navigates to this distinct page showing a parent-app phone mockup. From preview: "Post Now" button + "Back" link.
- `/announcements/drafts/:id` — draft pre-filled from HeyTalia's "Use draft" action. Auto-save active ("Last saved today at HH:MM AM"). Currently no such route.
- `/announcements/edit/:id` — spec treats edit as a separate URL from create. Currently codebase merges both into `/announcements/new`.

### 3b. Create/Edit form — sections missing

- **Website links section** (up to 3): two fields per entry — Link URL + Link description, "+ Add website link" button
- **File attachments section** (up to 3, 5MB each): drag-and-drop zone + "Add files" CTA button
- **Photo gallery section** (up to 12 photos, 3 as cover): drag-and-drop zone + "Add photos" CTA + thumbnail grid with cover toggle
- **Auto-save indicator**: "Last saved today at HH:MM AM" ticker on the page

### 3c. Detail page — modals missing

- **"Add Staff-in-charge" Modal**: triggered from Details tab "+ Add" button. Has staff autocomplete search, warning text, Cancel/Add buttons.
- **"Edit Enquiry Email" Modal**: triggered from Details tab "Edit" button. Radio options (OTP email / school email / other) + Cancel/Edit buttons (Edit disabled until selection changes).
- **Delete draft modal**: triggered from list kebab → Delete. Body: "Once you delete this draft, you will not be able to get it back again." Buttons: "No, not now" | "Delete draft".

### 3d. "Shared with you" tab

- Tab exists in spec with a "CREATED BY" column added to the table. Current implementation status unclear — verify whether this tab and its additional column are wired up.

- **Auto-save indicator** — same gap as Announcements

### 4d. Detail page — modals missing

- **"Edit Response" modal**: triggered from Responses tab "Edit Response" sub-link per student row. Fields: parent's response radio (Yes/No or Acknowledged/Not Acknowledged) + Comments textarea (500 char) + Cancel/"Update response".
- **"Edit Due Date" modal**: triggered from Details tab "Response Due Date and Reminders" section Edit button. Has warning banner ("Parents will be notified of the new due date"), new due date picker, auto-computed reminders, additional reminders radio. Cancel/Update.
- **"Delete Forever" with checkbox**: in OTHER ACTIONS section of detail page. Warning text + "I am sure I want to delete this form" checkbox + "Delete Forever" button disabled until checked.

## 5. GROUPS — Gaps (~25%)

Core functionality exists but some screens are missing.

### 5a. Edit Group page

Currently editing happens inline in the detail page. Spec defines a separate page with:

- Title input (120 char, pre-filled)
- Students section: "N students added", "+ Add Students" dropdown (Add manually / Upload via Excel), table with Name+ID, Class/Index, CCA(s), trash icon, "Delete All" link
- "Cancel" link | "Save" button (disabled until changes made)

### 5b. Add Students page

Spec defines a full page (not modal) with:

- Filter: "All Students" | "CCA" radio
- Search input: "Search student name or class name"
- Level dropdown, Form Class dropdown
- Results table: checkbox + Name (+ID) + Class/Index + CCA(s), select-all, pagination
- Bottom: Cancel | "Add N selected" (enabled when ≥1 checked)

### 5c. Share Group modal

Triggered from Details tab "Share Group" button. Contains:

- Staff search/autocomplete input
- Bullet list explaining what sharing grants (view/edit/add-delete/share)
- "Cancel" | "Share group" (disabled until ≥1 staff selected)

### 5d. Delete Custom Group modal

Triggered from Details tab "Delete Forever" button. Contains:

- "Delete custom group?" title
- Body: "Are you sure you want to delete '[Group Name]'?"
- Checkbox: "I understand that this action cannot be undone..."
- "Delete" button (disabled until checkbox ticked)

### 5e. Details Tab on Custom Group detail page

Spec defines a Details tab alongside Students tab with:

- Created on date/time by [StaffName]
- "Group shared with:" staff names list
- OTHER ACTIONS section: Edit Group button, Share Group button, Delete Forever button

## 8. HEYTALIA — Gaps (~10%)

### 8a. Conversation History modal

Clock icon (🕐) in toolbar should open a modal:

- Title: "Conversation History (N)"
- Scrollable list of past conversation sessions
- Edit button per session (to resume/rename)
- X to close

### 8b. Feedback modal

Speech bubble icon (💬) in toolbar should open:

- Title: "Share your feedback"
- 5-star rating widget
- Textarea for comments
- Cancel | Submit buttons

### 8c. File attachment flow

Paperclip icon (📎) in text input bar should trigger a file input. Flow not implemented.

### 8d. "Send email" modal

"Send email" action button on AI response bubbles should open:

- "Add email recipient(s)" modal
- Shows recent recipients as clickable pill chips
- Text input: "email@schools.gov.sg" placeholder, press Enter to add
- Cancel | Preview (disabled until ≥1 recipient)
- Preview step before send

### 8e. Inline edit mode on response bubbles

Edit (✏️) button on AI response bubble should transform the bubble into an inline rich text editor with B/I/A/≡ toolbar, Cancel + Confirm buttons.

### 8f. "Use draft" navigation

"Use draft" on announcement drafts should navigate to `/announcements/drafts/:id` (pre-filled edit form). "Use draft" on form drafts should navigate to `/consentForms/drafts/:id`. Both target routes are also missing (see sections 3a and 4b above).

### 8g. Thumbs feedback tag chips

👍 and 👎 buttons should expand an inline "Tell us more:" panel with clickable tag chips:

- 👍 tags: "Tone matched school style" | "Contains all key details" | "Concise and clear" | "Factually correct" | "Others"
- 👎 tags: "Tone doesn't match school style" | "Missing key details" | "Too wordy / unclear" | "Not factually correct" | "Others"
- X to close the panel

## 9. GLOBAL SHELL — Gaps

These are cross-cutting pieces that belong in `web/platform/` but aren't built yet.

### 9a. Footer

Spec defines a persistent footer:

```
© 2018 Ministry of Education, Singapore. All rights reserved.
Last updated on [date]           Help & Support ↗   Report Vulnerability ↗
```

- Help & Support → `https://go.gov.sg/pgsupport`
- Report Vulnerability → `https://www.tech.gov.sg/report_vulnerability`

### 9b. User profile dropdown (avatar menu)

The avatar circle in the nav should open a dropdown with:

- Displays `{staffName}`
- "View Profile" → `/account`
- "Switch to Admin View" (shuffle icon, only if admin role) → `/admin/...`
- "Notification Preferences" (mail icon) → `/notification-preferences`
- "What's New" (gift icon) → opens a changelog modal
- "Help & Support" (question circle) → external link
- "Logout" (orange, destructive) → ends session

### 9c. Network connectivity banner

Orange persistent banner: "You seem to have a bad network connection. Some functionality will be unavailable until your network is restored." Dismissible only via X. Appears on fetch failure detection.

### 9d. Web notification banner

Rendered from `configs.web_notification` when `enabled: true`, between `startDateTime` and `endDateTime`. Currently the feature flag is modelled but this banner component isn't built.

### 9e. Session timeout handling

`sessionTimeLeft` from session response (in seconds) should drive a warning when <5 min remain. On any 401 response → redirect to login/SSO.

### 9f. 404 page

Spec defines a branded page:

- Centered layout, PG logo (sad variant)
- "Page Not Found" heading
- "The page you're looking for could not be found."
- "Back to previous page" link (navigates `window.history.back()`)

Currently TanStack Router's `$.tsx` catch-all route may handle this but it's not the branded design above.
